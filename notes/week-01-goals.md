# Week 1 Goals

## 本周主题

建立 AI x Web3 的共同语言，并完成第一组可验证的 proof-of-work。

## 本周学习目标

1. 理解 AI 基础概念：LLM、prompt、workflow、agent、tool use、AI coding。
2. 理解 Web3 基础链路：账户、钱包、签名、交易、Gas、合约、测试网、区块浏览器。
3. 完成三个最小实践：一次 AI 工具实践、一次测试网交互、一个 AI x Web3 最小交叉实验。
4. 建立安全意识：权限、安全、人工确认、日志、验证材料和失败恢复。
5. 为 Week 2 的支付、身份、权限、安全隐私、治理协作等方向建立共同语言。

## 建议学习顺序

1. 先判断自己的已有基础：AI、Web3、编程都按新手路径处理。
2. 先补短板：用 learning agent 辅助理解 AI 和 Web3 基础词汇。
3. 完成两个方向的最小实践：一个 AI 工具任务，一个测试网或合约交互任务。
4. 再做最小交叉实验，把 AI 输出、人工复核、钱包确认、链上执行和验证记录放到同一条流程里。
5. 最后整理概念说明和过程记录，为后续项目方向讨论提供材料。

## 暂缓内容

这些内容先作为进阶方向，不作为 Week 1 新手路径的阻塞项：

- Foundry / Hardhat 工程化合约开发
- Safe、多签、ERC-4337、账户抽象
- LoRA / PEFT / 模型微调
- viem / wagmi 前端交互页
- 多 agent / skills / tracing 深度工程化

## 安全边界

- 不把私钥、助记词、API Key、密码、验证码写入仓库或聊天记录。
- 涉及签名、授权、转账、合约写入时，必须人工确认。
- AI 可以生成说明、脚本和检查清单，但不能替代钱包确认和安全判断。
- 所有链上实验优先使用测试网。
